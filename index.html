1. Introduction

[Draft: some figures may help to understand the text] The DISK project aims to help scientists to test and discover new hypothesis from existing datasets. Scientists define lines of inquiry that represent generic research questions, like the proteins associated to a type of cancer. The lines of inquiry may have multiple hypothesis associated to them, which define a particular instance or aspect of the line of inquiry (e.g., the EGFR protein is associated to colon cancer). Each hypothesis is represented as a connected graph, where the nodes are the different terms that compose the hypothesis and the edges represent the relationships among those terms.

Each of the statements represented in the hypothesis graph can be supported by the results obtained after running an experiment. In fact, the hypothesis has a confidence value, which varies depending on the results of the experiments run to test it. The provenance of the results is crucial to determine the confidence value associated to the hypothesis, and thus it needs to be captured as well.

A hypothesis may also be modified after reviewing the results of an experiment. The relationship among the different versions of the hypothesis (i.e., its evolution) is another key requirement to take into consideration.

In summary, we have four main requirements for representing hypotheses:

  1. Hypothesis element description, which provides the means to appropriately describe the contents of a hypothesis and their relationships;
  2. Confidence and provenance of the support results, which captures why a hypothesis has a certain confidence value, and how that value was obtained;
  3. Hypothesis evolution, which represents the different versions among the hypothesis and their relationships,
  4. Hypothesis metadata, which allows providing credit, responsibility and attribution to the creators and contributors of a given hypothesis.

In order to review and compare different existing vocabularies and models we have created a comparison framework that helps identifying the features and gaps of each of the approaches. The framework is described in the next section, followed by a summary of the different vocabularies and a discussion about which approaches would be the most appropriate to reuse.

2. Disk vocabularies

The Disk project defines four different vocabularies: an omics vocabulary, a hypotheis vocabulary, a neuro vocabulary, and a general purpose Disk vocabulary.

2.1 Omics vocabulary

The omics vocabulary introduces a set of terms for referring to biology terms that are referred to in hypothesis: proteins, patients, tissues, etc.

Browse the documentation of the vocabulary for more details.

2.2 Hypothesis vocabulary

The hypothesis vocabulary introduces a set of terms and properties for linking the terms included in a hypothesis.

Browse the documentation of the vocabulary for more details.

2.3 ENIGMA Hypothesis vocabulary

The ENIGMA Hypothesis vocabulary introduces a set of terms used in the DISK Hypothesis Reasoner for the neuroscience/neuroimaging domain: brain characterstic, patients, brain scan, etc.

Browse the documentation of the vocabulary for more details.

2.4 Disk vocabulary

The Disk vocabulary introduces the terms for creating hypothesis graphs and associate them to different lines of inquiry.

Browse the documentation of the vocabulary for more details.
